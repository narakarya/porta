Full Codebase Restructure — 3 Approaches

Pilih pendekatan yang paling cocok untuk Porta

A

Domain-Driven Split

Konsep: Reorganisasi berdasarkan domain bisnis — setiap domain (app, workspace, service, deploy, tunnel, terminal, gdrive) jadi module mandiri yang punya commands, DB repo, dan frontend components sendiri.

Pros

  • Navigasi sangat intuitif — cari "deploy"? buka folder deploy/
  • Ownership jelas per domain
  • Scaling natural — domain baru = folder baru
  • Backend & frontend aligned by domain

Cons

  • Cross-domain dependencies perlu careful wiring (app lifecycle depends on caddy, tray, db)
  • Shared utilities perlu dedicated shared/ layer
  • Biggest diff — paling banyak file moves
B

Layer-Based Split

Konsep: Reorganisasi berdasarkan layer teknis — commands/, db/, services/ (business logic), dan frontend: components/, hooks/, lib/, store/. Tiap layer di-split internal by domain tapi boundary utama adalah layer.

Pros

  • Familiar pattern — kebanyakan Tauri/React apps pakai ini
  • Tauri command registration tetap straightforward
  • Lebih kecil diff-nya dibanding Domain-Driven
  • Rust module system naturally fits layers

Cons

  • Related code scattered across layers (deploy command di satu folder, deploy UI di folder lain)
  • Kurang natural untuk reasoning about a feature end-to-end
  • Masih bisa jadi "big bucket" di tiap layer
C

Hybrid — Layer Backend + Domain Frontend

Konsep: Backend pakai layer-based (commands/, db/ sub-modules) karena Rust module system cocok. Frontend pakai domain-based grouping (features/ atau domain folders) + shared primitives. Best of both worlds.

Pros

  • Backend tetap idiomatic Rust — commands/mod.rs re-exports, db/repos terpisah
  • Frontend components grouped by feature — easy to find related UI
  • Shared UI primitives + hooks extracted ke common layer
  • Pragmatic — follows how each stack naturally organizes

Cons

  • Dua mental models — perlu tahu backend = by layer, frontend = by domain
  • Slightly more planning needed for the boundary

Rekomendasi gue. Rust module system works best with layer separation (commands/, db/). React components work best grouped by feature. Ini approach paling natural untuk Tauri app.
