Architecture — Tokens→Primitives→Features→Shell

A four-layer architecture that makes future redesigns cheap: tokens and primitives at the bottom (what a redesign changes), features and shell on top (untouched).

4 · Shell
Rail · domains · slots (rail/sidebar/main). IA = recompose slots.
3 · Features
workspaces · hosts · git · publish… = hooks (logic) + views (compose primitives). No raw styling.
2 · Primitives (ui/)
Button · Input · Card · Modal · Tabs · Badge · StatusDot · Popover · ListRow · EmptyState · Skeleton — built only on tokens.
1 · Tokens
surface/text/status colors · radius · spacing · type · motion (CSS vars / Tailwind theme). One source of truth.
A redesign changes
only layers 1–2
Features + shell inherit it for free — no file-by-file restyle.
